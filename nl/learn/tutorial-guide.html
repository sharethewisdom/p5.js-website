Een gids om bij te dragen met nieuwe p5.js leermiddelen.

geschreven door Tega Brain.

Wij nodigen educatoren en vrijwilligers uit om leermiddelen te maken. Het p5js project is erop gericht om inclusief te zijn en het programmeren toegankelijker te maken, en we zijn enthousiast als we allerlei aspecten van het ontwikkelen kunnen publiceren. Momenteel hebben we leermiddelen voor p5, voor algemene programmeer technieken en over hoe je kan bijdragen aan een open source project.

We verwelkomen bijdragen van leermiddelen, en dit stappenplan kan je daarbij helpen.

Aan de slag:

  • Kijk na dat het onderwerp waarover je wil gaan schrijven al niet reeds behandeld is. Er is een spreadsheet die een overzicht biedt in de voortgang van de ontwikkelde leermiddelen. Als je onderwerp erbij staat, kan je misschien daaraan meewerken. Laat ons iets weten dan.
  • Als je onderwerp er niet bij staat, beschrijf dan je voorstel in een paar zinnen en email het naar education@p5js.org.

Hoe je een p5js tutorial voorbereidt om te publiceren:

Als je tutorial klaar is om te publiceren, volg dan aub deze stappen om je inhoud klaar te maken voor de p5js website.

Maak de inhoud van je tutorial in een tutorial-naan.hbs bestand met deze basis structuur . Zoals je kan zien in dit bestand, moet het een hoofding bevatten zoals hieronder:

De map waarin je tutorial is opgeslagen zal in de 'tutorials' map van de p5js website geplaatst worden. Het bestand genaamd index.hbs is dep5.js tutorials startpagina, en het test-tutorial.hbs bestand is de test tutorial.

Alle inhoud moet in de: <section > </section> tags op de pagina gaan, met gebruik van de <h1> en <h2> hoofding tags en de <p> paragraaaf tags zoals je kan zien op detest tutorial pagina.

Als je tutorial afbeeldingen bevat, moet je die in de 'assets' map plaatsen van de p5js website, in src/assets/learn/test-tutorial/images zoals je hieronder kan zien.

Om code correct weer te geven in de html, gebruik de tag: 


<pre><code class="language-javascript">
Your code here!
</code></pre>

Embedden van p5.js schetsen

Je kan ter illustratie in je tutorial geanimeerde, interactieve, en bewerkbare voorbeelden insluiten. Je demonstraties moeten als p5.js schetsen worden voorbereid en kunnen op twee manieren in je tutorial worden verwerkt.

  1. Als het voorbeeld bewerkbaar moet zijn zoals in de documentatie paginas van de p5js site, dan moet de p5 schets met behulp van de p5js widget worden ingesloten. Volg deze gids over hoe je p5js schetsen moet insluiten met de widget geschreven door Toolness. Je kan zo'n resultaat ook zien op de test tutorial pagina.
  2. Als je voorbeeld geanimeerd en/of interactief moet zijn, maar niet bewerkbaar, dan moet je de schets in een iframe embedden, zoals hieronder beschreven.

Insluiten van een p5 schets met en iframe

Een iframe is als het maken van een venster in een webpagina waarin je een andere pagina kan verkennen. In dit geval zal het een venster zijn die de index.html pagina van je p5.js schets bevat.

Zet je p5 schetsen in de map /src/assets/learn van de p5js website, in een map met de naam van je schets zoals je kan zien in de schreenshot. Dit is waar je alle afbeeldingen en schetsen waarnaar je verwijst in iframes moeten worden opgeslagen.

In de onderliggende mappen van je voorbeelden moet er een sketch.js en een embed.html bestand zijn voor je schets.

Zorg dat je embed.html bestand juiste links bevat naar de p5 programma bibliotheken van de site. Als je bestandstructuur dezelfde is als hierboven, dan moet de link "../../../js/p5.min.js" zijn.

Je kan dan de index bestanden als iframes insluiten in het .hbs bestand waarin de inhoud staat. De embed code voor het iframe zal dan zo zijn:


<iframe src="http://p5js.org/assets/learn/tes-tutorial/embed.html" width="600" height="400">
</iframe>

Het stijlen van het iframe (dit kan direct in de post of in een styleheet):


<style> iframe{ border: none; } </style>

Hier vind je bijvoorbeeld een schets:

http://staging.p5js.org/assets/learn/test-tutorial/embed.html

En hier is het ingesloten in de p5 site met de volgende code:

http://staging.p5js.org/learn/test-tutorial.html

Eén ding dat we nog opmerken, is dat je de grootte van het iframe manueel moet bepalen, dus het werkt het beste als dingen een standaard grootte hebben.

Merk ook op dat de links naar de p5.js programma bibliotheken niet uitgaan van de .eps pagina met alle inhoud, maar in de html pagina die je schets weergeeft (in dit geval embed.html).

Meer informatie over het embedden van p5.js schetsen vind je hier.

Tot slot

Van zodra je klaar bent met het schrijven van je tutorial en je inhoud werd goedgekeurd, mag je je bestanden volgens de bovenstaande structuur aan je eigen kopie van de p5.js website git repository toevoegen. Je maakt dus een "Fork" van de repo, en daarin maak je een nieuwe branch waarop je je bestanden toevoegt. Dit sla je op met een commit die je online publiceert. Vervolgens kan je een pull request maken in de originele p5.js website git repo, zodat we je branch kunnen opnemen en je bijdrage kunnen publiceren!
